Accessible Mega Menu

A premium, framework-agnostic mega-menu navigation component. It features multi-column grid layouts, a robust accessible disclosure pattern, automatic horizontal positioning safe-guards, and a built-in industry SVG library. Hover over the utilities in the top-navigation above to experience it.

Documentation & Integration

1. Initial Setup

To use this module in your project, copy the relevant CSS (or SCSS/LESS), the Vanilla JS file (mega-menu.js), and the hidden <svg> sprite block deeply embedded at the bottom of the HTML page. You must include the SVG block on every page where the mega menu runs, ideally placing it inside a layout file footer.

To prevent grid clipping, your header container must be bundled logically. Give your navigation wrapper position: relative to mathematically frame the mega-menus safely. Second, you must apply overflow-x: hidden strictly on your layout's html and body tags to block animation drift.

<!DOCTYPE html>
<html lang="en" class="no-js"> <!-- The JS automatically strips this class -->
<body>
  <header style="border-bottom: 1px solid var(--mn-border);">
    <!-- The relative wrapper acts as the boundary wall for massive menus -->
    <div style="max-width: 1200px; margin: 0 auto; position: relative;">
      <a href="/">Brand.</a>
      <nav aria-label="Main Navigation" class="mega-nav" id="main-nav">
        <!-- Menu Items -->
      </nav>
    </div>
  </header>
  
  <!-- Paste the SVG System Sprite here -->
  <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">...</svg>
  
  <!-- Include the Navigation Controller -->
  <script src="mega-menu.js"></script>
</body>
</html>

2. Using the Icon System

This layout aggressively avoids heavy .woff2 font icons, exclusively utilizing a zero-byte inline stroke SVG system. The SVGs automatically assume the text color via currentColor.

To render an icon anywhere, assemble an empty SVG containing the .mega-icon utility class. Inside, point a <use> tag to the exact ID defined in your system sprite base (e.g. #icon-house, #icon-briefcase).

Check the Iconography reference below for a full visual library!

<a href="/home" class="mega-nav__panel-link">
  <!-- aria-hidden="true" tells screen readers to correctly skip the decorative shapes -->
  <svg class="mega-icon" aria-hidden="true">
    <use href="#icon-house"></use>
  </svg> 
  Roofing Services
</a>

3. Constructing Columns

The core sub-menus operate via native progressive CSS Grids. You can split any panel perfectly by passing a layout class explicitly onto the .mega-nav__panel layer. Everything wraps dynamically.

  • 2 Columns: Apply class="mega-nav__panel mn-grid-2"
  • 3 Columns: Apply class="mega-nav__panel mn-grid-3"
  • 4 Columns: Apply class="mega-nav__panel mn-grid-4"
  • Advanced Columns: Push an inline style tag directly like style="--mn-cols: 5;" on the panel layout object.

Down inside the grid framework, isolate groupings of links inside .mega-nav__category wrappers. You can slap a .mega-nav__category-title over any of them to create a labeled column section.

<div class="mega-nav__panel mn-grid-3" id="panel-services">
  <div class="mega-nav__panel-inner">
    <div class="mega-nav__panel-grid">
      
      <!-- Column Category 1 -->
      <div class="mega-nav__category">
        <h3 class="mega-nav__category-title">Trades</h3>
        <ul class="mega-nav__panel-list">
          <li><a href="#" class="mega-nav__panel-link">Plumbing</a></li>
          <li><a href="#" class="mega-nav__panel-link">Electrical</a></li>
        </ul>
      </div>
      
      <!-- Column Category 2, etc... -->

    </div>
  </div>
</div>

4. Mega Spread vs. Local Alignment

The behavior of how panels unfurl relates strictly to how wide their grids are.

  • Massive Mega Menus (3 or 4 columns): Tag your <li> line item wrapper with the .mn-mega utility class. This logic forces the massive payload to snap natively to the parent position: relative header wall, calculating padding flawlessly so they never slice off the boundary constraints.
  • Compact Local Dropdowns (1 or 2 columns): Exclude the .mn-mega tag. If the compact dropdown rests far to the right side of the screen, insert the .mn-align-right positional attribute alongside your grid definition instead. This traps it cleanly, forcing it to cascade downwards and leftwards back over the text safely.
<!-- Massive Centered Header-Spanning Layout -->
<li class="mega-nav__item mn-mega">
  ...
  <div class="mega-nav__panel mn-grid-4">...</div>
</li>

<!-- Local Safely Right-Aligned Compact Popover -->
<li class="mega-nav__item">
  ...
  <div class="mega-nav__panel mn-grid-2 mn-align-right">...</div>
</li>

5. Adding Featured "Promo" Blocks

You can spotlight Call-to-Action conversion funnels alongside links instantly by loading the custom .mega-nav__promo node directly inside the grid tier. It utilizes distinct -title, -text, and -btn sub-components to create high urgency aesthetic contrasts universally.

<div class="mega-nav__promo">
  <h4 class="mega-nav__promo-title">Need a quote?</h4>
  <p class="mega-nav__promo-text">Get an expert assessment entirely for free.</p>
  <a href="/contact" class="mega-nav__promo-btn">
    Estimate
    <svg class="mega-icon" aria-hidden="true" style="margin-left:auto; width:1em; height:1em;">
      <use href="#icon-arrow-right"></use>
    </svg>
  </a>
</div>

6. Accessibility & The No-JS Fallback

This widget acts firmly as a Disclosure Pattern. It aggressively bypasses faulty role="menubar" setups that cripple screen flow. Instead, accessibility devices tab across completely normal anchor objects, treating dropdown buttons contextually.

If JavaScript fails or is restricted locally, the menu falls backward onto a pure, CSS-driven cascading tree triggered via :hover and :focus-within listeners, provided the fundamental .no-js class exists on the foundational HTML shell element.
